5) Importante: evitar duplicados (sin tocar Program.cs)

Como Task Scheduler puede ejecutar al inicio de sistema y al iniciar sesión, puede intentar abrirlo 2 veces si coincide.

Solución rápida (código) dentro de FrmAsistencia_Load al inicio:

var current = Process.GetCurrentProcess();
var procesos = Process.GetProcessesByName(current.ProcessName);

if (procesos.Length > 1)
{
    Application.Exit();
    return;
}

📍 Dónde: primera línea de FrmAsistencia_Load.

✅ Esto evita 2 instancias.

6) Pregunta final que define si Task Scheduler basta o necesitas servicio

Para un sistema de marcación, lo típico es:

Si la PC siempre tiene usuario logueado → Task Scheduler basta.

Si la PC puede estar en pantalla de login y aún así quieres que corra → Task Scheduler con “al iniciar sistema” + “aunque no haya sesión” (lo que te puse).

Si quieres “nivel industrial” (sin UI, 24/7, sin depender de sesión) → Servicio Windows (más complejo).
